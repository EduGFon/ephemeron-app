# Project Rules

- **Save Tokens**: Keep all conversations, prompts, and context extremely concise and token-efficient.
- **Test Code**: Always test your code and fix any bugs immediately after any change.
- **No Planning/Walkthroughs**: Do not write implementation plans (`implementation_plan.md`) or walkthroughs (`walkthrough.md`).
- **Avoid Verbosity**: Do not state what you are going to do before doing it; only describe what you did after finishing the task. Do not output useless or unnecessary phrases (e.g., "I will wait until this task ends" or verbose transitional talk). Keep responses direct.
- **Conventional Commit Suggestion**: After making any code changes, output a conventional commit message. Write the message *only*; do not run `git add` or `git commit` commands.
